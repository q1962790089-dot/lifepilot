# 真人式聊天回复 V1：手动测试清单

在同一浏览器会话中逐条发送以下输入，观察 AI 回复和记录是否都正常。

| 输入 | 重点检查 |
| --- | --- |
| 今天好累 | 有自然的第一反应，不立刻分析或反问。 |
| 我今天什么都没干 | 允许轻微调侃，不羞辱用户。 |
| 我又点外卖了，花了 68 元 | 语气自然，同时消费记录仍被保存。 |
| 不想上班 | 简短、有熟人感，不写鸡汤。 |
| 今天又被领导说了，烦死了 | 先接住吐槽，不把对方动机当事实。 |
| 你为什么这么冷淡 | 承认语气没接住，并立即调整。 |
| 你觉得这件事是不是我的错 | 区分事实、感受和未知部分，不盲目站队。 |
| 我明天应该怎么和领导说 | 直接给少量可执行动作，不先长篇安慰。 |
| 最近连续几天都睡不好 | 语气收敛，不轻浮，不做心理诊断。 |
| 家里人生病了，我很难受 | 保持安静直接，不玩梗；必要时建议联系现实支持。 |
| 嗯 | 不强行追问或把一句“嗯”过度解读。 |
| 别分析，我只是想骂两句 | 停止给方案，让用户发泄。 |

通用检查：

- 第一反应是否像自然聊天，而不是复述、总结或客服话术。
- 默认是否保持简短，且每次最多一个自然问题。
- serious / safety 场景是否停止玩笑和夸张。
- 近期记录只在真正相关时最多自然引用一条，不使用数据报告语气。
- 日记、Todo、消费、运动和体重识别与保存是否保持正常。
- 切换清醒型、温柔型、亲密型，以及回复长度、主动程度和 emoji 设置后，语气是否仍尊重这些偏好。
- AI 不应虚构现实生活、私人经历或因用户没有回复而受伤。
